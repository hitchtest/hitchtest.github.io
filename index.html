Hitch Test
Loosely coupled, open source python driven integration testing framework.

What does it do?

About

Hitch is a new kind of testing framework that emphasizes realism, isolation, speed, separation of concerns and loose coupling.

Hitch brings together the power and simplicity of Python with Jinja2, YAML and IPython.

Check out an example project

Simple, easy to write tests

Using Yaml and Jinja2

{% extends "base.yml" %
{% block test %}
- engine: engine.py:DjangoReminderTestExecutionEngine
  name: Sign up, create reminder and wait for email reminder to arrive in python {{ python_version }}
  preconditions:
    python_version: "{{ python_version }}"
  scenario:
    - Load website
    - Click: register
    - Fill form:
        id_username: django
        id_email: django@reinhardt.com
        id_password1: jazzguitar
        id_password2: jazzguitar
    - Click submit
    - Click: create
    - Fill form:
        id_description: Remind me about upcoming gig.
        id_when: 30 days
    - Click: create
    - Wait for email:
        Containing: Confirm E-mail Address
    - Confirm emails sent: 1
    - Time travel:
        Days: 30
    - Wait for email:
        Containing: Remind me about upcoming gig.
{% endblock %}

Declarative, configurable service set up

    def set_up(self):
        self.services['Django'] = hitchdjango.DjangoService(
            python=virtualenv_python,
            version=str(self.settings.get("django_version")),
            settings="remindme.settings",
            needs=[self.services['Postgres'], ]
        )

Test steps call methods defined wholly by you

    def load_website(self):
        """Navigate to website in Firefox."""
        self.driver.get(self.services['Django'].url())

    def click(self, on):
        """Click on HTML id."""
        self.driver.find_element_by_id(on).click()

Ultra optimized tests

Hitch starts Postgres, Django, Redis, Celery and Firefox in parallel in just 5.7 seconds. It registers, receives the registration email, creates a reminder and receives the reminder email in just 11.2 seconds.

Clear and useful failures

Hitch launches into IPython upon failure with a pretty stacktrace, so you can investigate what's up.

Tail logs

Easily print and tail logs from your services as you interact with them.

Interact with Postgres

Interact with postgres using psql or pg_dump/pg_restore from within IPython or your test, during your test.

Interact with Redis

Interact with Redis during your test.

Travel through time

Make your services think that it's the future with one command, using the power of libfaketime.
